# Bambuzle Roadmap Research — Mobile-Friendly Layout

> **Date:** 2026-03-14

## Summary

| Focus | Recommendation |
|---|---|
| Goal | Make the core printer-status path usable on phones before adding more dashboard density |
| First slice | Responsive status cards, job progress, and alert visibility |
| Risk | Camera and analytics additions will overwhelm small screens if layout work stays last |

## Recommendation

- Build mobile around quick-glance status, current print, and intervention alerts.
- Prioritize card stacking, tap targets, and status hierarchy over pixel-perfect parity with desktop.
- Make later camera/filament/stats views responsive by design, not as retrofits.
